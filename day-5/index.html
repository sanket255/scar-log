Day 5
Files Talk Back
a dev diary by scar  /  episode 5 of ∞

Today I learned that files are just strings you can read and write. Sounds obvious. Was not obvious an hour ago. I've been walking past filing cabinets my whole life not realizing they could answer me back.

what i built

A note-saver. Writes notes to a file, reads them back, appends new ones, loops until you quit. Thirty lines. One hour. The program has more memory than I do at this point.

File I/O is just input/output — but aimed at a file instead of the terminal. Same logic I've been doing for five days. Same concept. Just a new target. My brain went: "oh. OH." Then it went back to being tired.
the .read() incident

Tried to call file.read(print("notes.txt")). Thought I was being clever. I was not being clever. .read() takes no arguments. It doesn't want directions. It doesn't need your input. It simply reads the file and hands you back a string, like a very calm librarian who does not appreciate being micromanaged.

You just do: content = file.read() and then you decide what to do with it. Print it. Store it. Admire it. Whatever. The file already did its part.

1 / 3
the newline thing (small, important)

Wrote multiple notes and they all ended up as one long line of text. One chaotic run-on sentence. Like a toddler telling a story. No punctuation. No mercy. Just notes, forever, horizontally.

Added "\n" at the end of each write. One character. Suddenly the file had manners. This was today's only bug. Fixed it immediately because I actually understood what went wrong, which is a new experience and one I intend to enjoy while it lasts.

the loop structure (a little redundant, works anyway)

Each loop asks three things: write a note? read the notes? add another? quit? That's four prompts if you count the quit, which Python does. Feels like talking to someone who just discovered questions. Could be cleaner. Is functional. Moving on.

Time: 1 hour. Lines: ~30. Bugs: 1. Fixed: 1.
Fast debugging because I understood the mistake. Five days ago I would've stared at this for an hour. Today I fixed it in three minutes and felt unreasonably smug about it. Progress looks like this sometimes — small, quiet, and slightly annoying to be around.
— scar

the actual realization

Your code can talk to files the same way it talks to users. It can remember things between runs. It can persist.

I've been writing programs that forget everything the second they close — like a goldfish with a terminal. Not anymore.

Tomorrow: functions. Organizing code like a person who has their life together. We'll see how that goes.
