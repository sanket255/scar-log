Day 12
Real Data From
the Sky
a dev diary by scar  /  episode 12 of ∞

Built an app today that fetches actual weather data from actual cities on the actual planet. Delhi. Mumbai. Indore. Quito. Real temperatures. Real humidity. Real conditions. My Python script asked the sky what was happening and the sky answered. This is not a drill.

the nested dictionary problem

Started confused about why data['temp_c'] wasn't working. The data wasn't flat — it was a dictionary inside a dictionary, like a box inside a box, which requires two sets of bracket notation to open.

data['current']['temp_c'] — get into current first, then get temp_c from inside it.

And condition? That's a dictionary inside current. Three levels deep. data['current']['condition']['text']. Russian nesting dolls, but for weather data. Once you see it, it's obvious. Before you see it, it just looks like a personal attack.
the honesty section (i peeked)

Got stuck on the structure and looked at example code instead of figuring it out. Not full copy-paste — just the shape of the thing. Still counts as cheating a little. Not proud of it. Next time I'll sit with the confusion longer before looking. The confusion is where the learning lives. I short-circuited it. Won't do that again.

the bugs (three of them, all fixed)

Bug 1 — tried to print condition as a string. It's a dictionary. You have to go one level deeper for ['text']. The API gives you an object, not a sentence. Rude, but understandable.

Bug 2 — loop started with city = 0 then immediately checked while city != "quit". A number will never equal the string "quit", so the loop ran forever before even asking for input. Classic. Fixed by asking first, then looping.

Bug 3 — typo in ConnectionTimeout. It's still there. I caught it. I am choosing to address it tomorrow. This is called technical debt and I am accruing it deliberately.

what this actually means now

Twelve days ago: couldn't add two numbers without help. Today: fetching live weather data from four cities, parsing three levels of nested JSON, handling API errors, building two clean functions that work together. The internet is now a resource I can tap into from Python. That changes what's possible. Completely and permanently.
