Day 10
Quick and Dirty
a dev diary by scar  /  episode 10 of ∞

Built a quiz game in 20 minutes today. It is not pretty. It is not clean. It has five nearly identical functions that a computer science professor would use as a warning to students. It works perfectly.

the code (a confession)

Five questions. Five functions. Each one a copy-paste of the last with a different question and a different answer hardcoded inside. Fifty lines where ten would do. Genuinely terrible practice. I know this. I knew it while writing it. I wrote it anyway because I had thirty minutes and a working program beats a clean skeleton every single time.

I could've done this instead — one function, a loop, a list of questions: clean, elegant, reusable, ten lines total. I did not do that. I did the fast version and I am at peace with this decision.
fast & messy vs slow & clean

This is the actual tension in coding that nobody warns you about. Fast and messy — get it working now, fix it later, ship the thing, hit the deadline. Slow and clean — get it right immediately, future-you will be grateful, present-you misses the event.

For a thirty-minute task under time pressure? Fast and messy wins. For production code that six other people will read? Clean wins. Knowing which situation you're in — that's the actual skill. Today I was in a thirty-minute situation. Chose correctly.

the numbers (honest edition)

Time: 20 minutes of actual coding. Lines: 50+, most of them redundant. Questions: 5. Code quality: good acceptable terrible. Functionality: perfect.

That last line is the only one that mattered today. The program runs, the questions ask, the answers check, the score counts. Every requirement met. Every line of code an affront to elegance. Completely fine.

what day 10 actually means

Ten days ago I didn't know what a variable was. Today I wrote bad code on purpose — which means I understand good code well enough to know what I was sacrificing. You can only knowingly write messy code if you know what clean looks like. That's not nothing. That's actually a lot.

Speed without understanding = useless. Understanding without speed = procrastination. Today I needed speed. Got it. Knew the cost. Paid it willingly. That's the trade-off. That's the whole job sometimes.
— scar
