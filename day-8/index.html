Day 8
The Journal Lives
a dev diary by scar  /  episode 8 of ∞

Built a real app today. Not practice. Not a "toy project." Something I could actually use — saves notes, reads them back, searches them. Finished in 32 minutes. Which feels suspicious, considering yesterday existed.

what changed overnight

Yesterday: 2 hours fighting main(). Today: wrote the whole thing in minutes. Same idea. Same structure. Just no resistance this time.

The code didn't fight back. Which probably means I'm finally starting to understand what I'm doing. Took eight days. Worth it.
the structure (kept it boring on purpose)

One loop. One question. Four options: save, read, search, quit. No extra logic. No clever tricks. Just: ask → decide → call function → repeat. That's the whole app. Which is slightly annoying, considering how complicated I made yesterday's version.

1 / 3
the search (the obvious solution)

Loop through the file. Check if the keyword exists in the line. Print it. That's it. No fancy data structures. No algorithm. Just the obvious solution that I almost didn't use because I was looking for something smarter.

Turns out "simple" only works after you stop trying to be impressive. The impressive part was already done — it's called "building the whole app."
32 minutes vs 2 hours

Same concepts. Same file structure. Same function pattern. Yesterday it took 2 hours and still didn't work right. Today it took 32 minutes and works cleanly. The difference isn't talent. It isn't even practice. It's the overnight processing. Sleep ran the compiler in the background. I just showed up for the output.

That's the part nobody tells you about learning to code — some of it happens when you're not coding. You just have to trust the process enough to actually stop for the night.

— scar

the actual realization

Yesterday I was figuring out how things work. Today I just… used them.

Repetition doesn't make things easier. It makes them obvious. And once something feels obvious, you stop fighting it.

That's when things start to move.
