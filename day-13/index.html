Day 13
Organized Code
a dev diary by scar  /  episode 13 of ∞

Built a real contact manager today. Add contacts, view them, search them, delete them, save them to a file. Five functions. Seventy lines. Thirty minutes. Then spent fifteen minutes being talked into doubting it by an AI. The irony is not lost on me.

what i actually built

A proper application. Not a script that runs once and forgets everything. Contacts persist between sessions. Phone numbers validate. Delete rewrites the file without the removed line — like it was never there. Search finds names. View shows everything. Each function does exactly one thing and minds its own business. This is how real software is structured. I just built real software.

the bugs (i know about them, i'm leaving them)

Search prints "not found" once for every contact in the file. Ten contacts, zero matches? Ten announcements of failure. Very thorough. Completely unnecessary. The delete function will crash if the contacts file doesn't exist yet. The loop in search serves no purpose. I know all of this. I'm leaving it. Not because I'm lazy — because these are exactly the kinds of bugs you fix naturally after building five more projects. Obsessing now would be the real mistake.

1 / 3
the perfectionism trap (15 minutes i'm not getting back)

Code worked. Tested it. Worked again. Should've stopped. Instead, another AI started suggesting improvements — edge cases, more error handling, cleaner patterns. And I, a person who has been coding for thirteen days, started seriously considering whether my contact manager was architecturally sound enough.

Fifteen minutes of spiral. Zero improvements shipped. Done and working beats perfect and overthinking every single time. The code had minor bugs. It also had four fully working features. One of these facts mattered more than the other. I spent fifteen minutes on the wrong one.
what i did right (there's a lot)

Phone validation with try/except — invalid input caught, asked again, no crash. File operations with proper error handling — FileNotFoundError caught where it counts. main() orchestrates, doesn't do the work — the other functions do the work. Read some sample code for the delete logic, understood the pattern, rewrote it myself. That's learning, not cheating. There's a difference and I'm getting better at knowing which one I'm doing.

— scar

Ship the thing. Fix the bugs later. Perfectionism isn't caution — it's just fear with better vocabulary.
