Day 17
Objects Are Real
a dev diary by scar  /  episode 17 of ∞

Built my first real class today. Not understanding it in theory — everyone understands it in theory. Actually using it. Made four people from one blueprint. Called methods on them. Got different results from the same code. The penny dropped. It took 17 days but it dropped.

what a class actually is

A blueprint. A template. A recipe before you make the cake — except once you have the recipe you can make infinite cakes, each one with different flavours, and they're all still cakes. The class defines the shape. The object is the actual thing.

class People: — here's the blueprint.

p1 = People("sanket", 19, "Mumbai") — here's a person made from it.

p2 = People("riti", 18, "mumbai") — here's another one.

Same blueprint. Completely different people. This is the whole idea.
what __init__ does (it's not scary)

Runs automatically the moment you create an object. You pass in the values, __init__ receives them, attaches them to the object with self. self just means this specific object I'm creating right now — not the class, not some other object, this one. self.name = name means "this person's name is whatever was passed in." Simple once you stop treating it like a keyword with mystical powers.

the method thing (same code, different results)

Added an info() method. Called p1.info() — got Sanket's data. Called p2.info() — got Riti's data. Same method. Same line of code. Completely different output. That's not a trick. That's the entire point of objects. The method belongs to whoever's calling it.

sanket is 19 and live in Mumbai
riti is 18 and live in mumbai
jon is 23 and live in san F
krsna is 37 and live in delhi

Four objects. One blueprint. Twenty lines of code. This output used to require four separate functions or a lot of manual repetition. Now it's a loop and a method call. Organization changes everything.
why oop exists (finally makes sense)

Before: functions scattered everywhere, data floating around separately, trying to remember what connects to what. Now: data and the functions that use it live in the same place. The contact manager, the weather app, the quiz game — all of them would be cleaner as classes. I can see it now. Couldn't see it three days ago. That's what foundations do — they make the stuff you already built suddenly look different.

the actual realization

Created four people from one blueprint. Finally understand what "object" means.

OOP isn't magic. It's just putting related things in the same place so they stop getting lost. Data that belongs to a thing lives inside that thing. Functions that belong to a thing go inside that thing.

That's it. 17 days to get here. Going to sleep now.
