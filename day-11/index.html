Day 11
APIs Are Real
a dev diary by scar  /  episode 11 of ∞

Learned today that the internet isn't magic. It's just data being fetched from somewhere by something. You can be the something. This information has changed me personally.

what i built (yes, really)

A Kanye West quote fetcher. Pulls random quotes from a real API, shows them, saves favourites to a file, loops until you quit. Somewhere right now there is a server responding to my Python script with Kanye's thoughts. This is the world we live in. I built something that talks to it.

the autocomplete incident (embarrassing, instructive)

VS Code kept offering to finish my sentences and I kept saying yes. Like accepting every suggestion from a very confident stranger who may or may not know what they're talking about.

First version: data = requests.json() — autocomplete's idea. Wrong.

Autocomplete also added ConnectTimeout exception handling that I copy-pasted without knowing what a timeout was, what it connected to, or why anyone would want to catch it. Technically functional. Completely hollow. That's not learning. That's transcription.

Stopped. Deleted everything. Rewrote from scratch without touching a suggestion. Took twice as long. Understood every single line. Autocomplete is a tool, not a teacher — and I'd been using it as a substitute for thinking.

1 / 3
what an api actually is (finally)

requests.get(url) — asks a URL for data. Gets a response object back. .json() — turns the response into a Python dictionary. .status_code — 200 means it worked. Anything else means something went wrong and you should probably look concerned. Three things. That's the whole API interaction. I'd been treating this like dark magic for eleven days when it was just a very organised letter exchange.

the logic bug (and vs or)

Loop condition was: if ask != "y" or ask == "quit": break — which broke on basically any input because one of those conditions is always true no matter what you type. The or was letting everything through. Changed to and. Both conditions now have to be true simultaneously to matter. AND means both. OR means either. This should be obvious. It was not obvious until I broke a loop with it at 11PM.

Final version: fetches quotes, catches errors, saves favourites, loops cleanly. Ugly code. Every line mine. Every line understood. Somewhere a server is waiting for my next requests.get() and I know exactly what will happen when I send it. That feeling is new. I like it.
— scar

the actual realization

Spent 2 hours learning what autocomplete was stealing from me.

Every suggestion I accepted was a concept I didn't have to understand. Every line I let it write was a line I couldn't explain later. Fast code. Empty hands.

Write it yourself. Even when it's slower. Especially when it's slower.
