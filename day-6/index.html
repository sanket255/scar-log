Day 6
Return to Code
a dev diary by scar  /  episode 6 of ∞

Came back today with a fever and approximately 60% of a functioning brain. The remaining 40% was busy being confused about return. We made it work anyway. Medicine, spite, and sheer stubbornness.

attempt #1: the crime scene

Tried to build a calculator with functions. First attempt looked like this: no parameters, all conditions crammed inside every function, all four functions called at once regardless of what the user wanted. A calculator that did everything simultaneously and therefore nothing correctly. Like asking four people for directions and following all of them.

The core confusion: I had no idea what return actually did. I thought functions just… ran things. Did stuff. Existed. Turns out they're supposed to hand data back when they're done. Like an employee who actually gives you the report instead of just reading it to themselves and walking away.
the hunt (and the click)

Read examples. Read documentation. Did not understand immediately. Then it landed: return sends data back to whoever called the function. Not print. Not store. Return. The function takes numbers, does the thing, hands the result back. What you do with it after is your business.

1 / 3
i built this calculator four times today

Not because I enjoy suffering — though the data suggests otherwise — but because each rebuild taught me something the last one couldn't. Functions need parameters. Each should return, not print. One main function routes to the right one. Store the result. Then print it.

By attempt four it was clean. Tidy. Logical. The kind of code that looks obvious after you've written it wrong three times.

print vs return (the thing nobody explains clearly)

print — shows something on screen, code keeps going, data goes nowhere useful. return — passes the value back, function stops, now something else can use it.

One is for displaying. One is for passing data between functions. Completely different jobs. Same vibes. Zero overlap. Took me four rebuilds and a fever to really absorb this. Not my proudest timeline. Accurate one though.

Bonus: added division-by-zero error handling. On my own. Didn't look it up. Just thought — what if someone types 0? — and handled it. Small thing. But five days ago I wasn't thinking about edge cases. I was thinking about whether the code would run at all. That's the shift. Quiet but real.

the actual realization

Day 5 — files talk back. Day 6 — functions talk back too.

Data flows. Functions take input, process it, return output. That's not just today's lesson — that's the foundation for everything next.

If the concepts click when the brain is running at 60%, the foundations are actually solid. Fever certified. No notes.
