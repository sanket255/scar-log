Day 7
Functions Meet Files
a dev diary by scar  /  episode 7 of ∞

Built something today that looked simple on paper. Which, historically, is how I know it’s about to go wrong.

what i thought i was building

A note manager. Write notes. Read notes. Loop until you quit. Functions + files. Two things I already "understood."

Which means, naturally, I made it way more complicated than it needed to be.

the mess (self-inflicted)

Started with one variable doing everything. date was handling input, decisions, life choices — all of it.

Conditions were checking the wrong thing. Flow made no sense. The code technically existed, but it wasn't… thinking.

This is a pattern now: I don’t fail because something is hard. I fail because I try to be clever before being clear.
1 / 3
the fix (boring, effective)

One variable. Clear options. No overthinking.

Ask user what they want. Call the function. Loop.

That’s it. That was the whole solution. Not new knowledge. Just cleaner thinking.

if ask == "s" or "w":

This line worked. Which was the problem. It always worked.

Turns out "w" is always true. So the condition was basically:

"if anything happens, do this."

Fixed it:
if ask == "s" or ask == "w":

One extra comparison. Completely different logic.

2 / 3
the actual realization

Functions handle one thing. Files remember things. Main decides what happens.

That’s the system. Not complicated. Just easy to mess up if you don’t respect it.

Combining concepts isn’t harder. It just exposes whether you actually understand them.

Today, I mostly did.

That’s enough.

— scar
